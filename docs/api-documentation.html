Backend API

Documentation complète de l'API REST du réseau Alumni de l'ENSPM. Toutes les routes, paramètres, corps de requête et exemples de réponse nécessaires à l'intégration frontend.

Base URL http://your-domain.com/api
Framework Laravel 11 + Sanctum
Format JSON (application/json)
Auth Bearer Token
📖

Introduction

Vue d'ensemble

Cette API alimente la plateforme alumni de l'ENSPM (École Nationale Supérieure Polytechnique de Maroua). Elle gère les profils utilisateurs, les publications sociales, les offres de stage/emploi/formation, les forums, un système de commentaires/likes, des signalements de contenu et une messagerie en temps réel.

ROUTES PUBLIQUES

Seulement 5 routes ne nécessitent pas d'authentification : POST /login, POST /forgot-password, POST /verify-otp, POST /resend-otp, POST /reset-password

ROUTES PROTÉGÉES

Toutes les autres routes requièrent un header Authorization: Bearer {token}. Le token est obtenu via POST /login.

🔐

Authentification

L'API utilise Laravel Sanctum avec des tokens Bearer. Après connexion, incluez le token dans chaque requête protégée.

Header requis pour toutes les routes protégées

Authorization: Bearer votre_token_ici
Content-Type: application/json
Accept: application/json
📦

Format des réponses

Toutes les réponses suivent une structure JSON unifiée. Le champ type indique toujours le résultat de l'opération.

200/201 Succès
{
  "type": "success",
  "status": 200,
  "message": "Opération réussie",
  "data": { // présent si non vide
    // données retournées
  }
}
422 Erreur de validation
{
  "type": "validation_error",
  "status": 422,
  "message": "Validation Error",
  "data": {
    "email": "Le champ email est requis",
    "password": "..."
  }
}
403/404 Erreur
{
  "type": "error",
  "status": 403,
  "message": "Non autorisé"
}
200 Réponse personnalisée (ex: OTP expiré)
{
  "type": "expired",
  "status": 403,
  "message": "OTP expired. Please try again"
}

Auth & Réinitialisation de mot de passe

Authentifie l'utilisateur et retourne un token Sanctum.

POST /api/logout Déconnexion 🔒 Auth ▼

Révoque le token d'accès courant. Aucun corps de requête requis.

200 Réponse
{ "type": "success", "status": 200, "message": "Logout successfully" }
POST /api/forgot-password Demander un OTP de réinitialisation ▼

Envoie un code OTP à 6 chiffres valable 5 minutes par email.

ParamètreTypeDescription
emailrequisstring (email)Email du compte à réinitialiser
200 Réponse
{ "type": "success", "message": "OTP sent successfully", "data": { /* objet user basique */ } }
POST /api/verify-otp Vérifier l'OTP reçu par email ▼

Vérifie que le code OTP est correct et non expiré. Si le code a expiré, le type sera "expired" avec le code 403.

ParamètreTypeDescription
emailrequisstringEmail du compte
otprequisstring (6 chiffres)Code OTP reçu par email
POST /api/resend-otp Renvoyer l'OTP ▼

Génère et renvoie un nouvel OTP. Un OTP doit avoir été demandé au préalable via /forgot-password.

ParamètreType
emailrequisstring (email)
POST /api/reset-password Réinitialiser le mot de passe ▼

Réinitialise le mot de passe après vérification de l'OTP. Si logout est true, tous les tokens existants sont révoqués.

ParamètreTypeDescription
emailrequisstringEmail du compte

Retourne les publications paginées. Inclut l'utilisateur auteur et les médias associés.

Query paramTypeDescription

  }

GET /api/posts/{id} Voir une publication 🔒 Auth ▼

Retourne le détail d'une publication avec ses médias et son auteur.

POST /api/posts Créer une publication 🔒 Auth ▼

Envoyez en multipart/form-data. Au moins content ou medias doit être fourni.

ParamètreTypeContrainte
contentoptionnel*stringTexte de la publication
medias[]optionnel*file[]Images ou vidéos, max 30MB chacun

* Au moins l'un des deux est obligatoire.

POST /api/posts/{id}/update Modifier une publication 🔒 Auth ▼

Si de nouveaux médias sont fournis, tous les anciens médias sont remplacés. Seul l'auteur peut modifier.

GET /api/posts/{id}/delete Supprimer (suspendre) une publication 🔒 Auth ▼

La suppression est une suspension logique (status → "suspended"). La publication disparaît du fil mais les données sont conservées.

Commentaires

Le système de commentaires est polymorphique : il fonctionne sur les posts, stages, formations et offres d'emploi. Les commentaires supportent une seule réponse de profondeur (pas de réponses imbriquées).

GET /api/{resource}/{id}/comments Lister les commentaires d'une ressource 🔒 Auth ▼

Remplacez {resource} par : posts, internships, trainings, ou job-offers.

200 Réponse
{
  "data": {
    "comments": [
      {
        "id": 1,
        "body": "Super post!",
        "parent_id": null,
        "user": { /* auteur */ },
        "replies": [
          { "id": 2, "body": "Merci!", "parent_id": 1, "user": {/*...*/} }
        ]
      }
    ],
    "pagination": { "total": 10, "per_page": 20, "current_page": 1, "last_page": 1 }
  }
}
POST /api/comments Créer un commentaire ou une réponse 🔒 Auth ▼

Pour un commentaire : fournissez commentable_type et commentable_id.
Pour une réponse : fournissez uniquement parent_id (l'ID du commentaire parent).

ParamètreTypeDescription
bodyrequisstringContenu du commentaire
commentable_typesi pas de parent_idstringpost, internship, training, job_offer
commentable_idsi pas de parent_idintegerID de la ressource
parent_idpour répondreintegerID du commentaire parent

⚠️ Les réponses imbriquées (répondre à une réponse) ne sont pas autorisées. Le serveur retourne une erreur 422.

POST /api/comments/update Modifier un commentaire 🔒 Auth ▼
ParamètreType
idrequisinteger
bodyrequisstring
DELETE /api/comments/{id} Supprimer un commentaire 🔒 Auth ▼

Seul l'auteur du commentaire peut le supprimer. La suppression est définitive.

Chat & Messagerie

Module de messagerie en temps réel supportant les conversations privées (1-to-1) et les groupes.

Structure d'un Chat

{
  "id": 1,
  "type": "private", // ou "group"
  "chat_title": "Simple Chat",
  "created_by": 1,
  "members": [{ "member_id": 1, "member_role": "admin" }],
  "last_message": { "message": "Bonjour", "created_at": "..." }
}

Retourne les conversations privées de l'utilisateur, triées par dernière activité.

POST /api/chats Démarrer une conversation privée 🔒 Auth ▼

Retourne les groupes dont l'utilisateur est membre.

Le créateur devient automatiquement admin du groupe.

ParamètreType
