Admin API

Introduction — Espace Admin

Vue d'ensemble

Ce document couvre exclusivement les nouvelles routes ajoutées dans cette version du projet. Il se divise en deux parties :

Routes Admin (/api/admin/*)

Réservées aux utilisateurs avec role = admin. Accès refusé (403) pour tout autre rôle. Gèrent les utilisateurs, les stages et les annonces de la plateforme.

Nouvelles routes Plateforme

Accessibles à tous les utilisateurs authentifiés. Incluent la lecture des annonces actives (/announcements) et la découverte des membres (/networking/members).

🔐

Préfixe & Contrôle d'accès

Toutes les routes admin sont protégées par deux couches de sécurité appliquées dans routes/api.php.

Double middleware sur toutes les routes /api/admin/*

Route::middleware(['auth:sanctum', 'role:admin'])->prefix('admin')->group(...);

1. auth:sanctum — vérifie la présence du Bearer Token
2. role:admin — vérifie que l'utilisateur a le rôle admin

Header à inclure dans chaque requête

Authorization: Bearer {token_admin}
Content-Type: application/json
Accept: application/json
401 — Non authentifié

Token manquant ou expiré. L'utilisateur doit se reconnecter via POST /api/login.

403 — Accès refusé

Token valide mais l'utilisateur n'a pas le rôle admin. Ne jamais exposer ces routes au frontend public.

📦

Format des réponses

Identique au reste de l'API : structure JSON unifiée avec les champs type, status, message et data.

200/201 Succès
{
  "type":    "success",
  "status":  200,
  "message": "Opération réussie",
  "data":    { /* données retournées */ }
}
422 Erreur de validation
{
  "type":    "validation_error",
  "status":  422,
  "message": "Validation Error",
  "data": {
    "email": "Déjà utilisé"
  }
}

Pour l'import en masse (bulkStore, import), le champ data contient deux compteurs : created (nombre de lignes importées) et skipped (tableau des lignes ignorées avec la raison).

Gestion des utilisateurs

Préfixe : /api/admin/users. Permet de créer des comptes, d'importer en masse depuis un fichier Excel/CSV, et de supprimer des utilisateurs. À la création, un mot de passe est généré automatiquement et envoyé par email.

GET /api/admin/users Lister tous les utilisateurs ⬡ Admin ▼

Retourne tous les utilisateurs paginés (30 par page), sans filtres. Inclut tous les rôles.

200 Réponse
{
  "type": "success",
  "data": {
    "data": [
      {
        "id": 1,
        "name": "Jean Dupont",
        "email": "jean@enspm.cm",
        "phone": "690000000",
        "role": "user",
        "domain": "Génie Civil",
        "graduation_year": 2022,
        "is_worker": true,
        "status": true,
        "created_at": "2026-01-15T10:00:00.000000Z"
      }
    ],
    "next_page_url": "http://your-domain.com/api/admin/users?page=2",
    "prev_page_url": null
  }
}
POST /api/admin/users Créer un compte utilisateur ⬡ Admin ▼

Crée un nouveau compte, génère automatiquement un mot de passe aléatoire (9 caractères) et envoie les identifiants par email à l'utilisateur.

Corps de la requête (JSON)

ParamètreTypeContrainteDescription
namerequis string – Nom complet
emailrequis string (email) unique Adresse email (doit être unique)
phonerequis string unique Numéro de téléphone (doit être unique)
rolerequis string – Rôle : user ou admin
domainoptionnel string – Domaine de spécialisation (ex : Génie Civil)
graduation_yearoptionnel integer 4 chiffres, ≥ 1900 Année de diplôme
is_workeroptionnel boolean – L'utilisateur est-il en activité professionnelle ?
200 Réponse succès
{
  "type": "success",
  "message": "Account created",
  "data": {
    "id": 42,
    "name": "Marie Curie",
    "email": "marie@enspm.cm",
    "role": "user"
    // mot de passe NON retourné — envoyé par email uniquement
  }
}

⚠️ Le mot de passe généré est envoyé uniquement par email à l'utilisateur. Il n'est jamais retourné dans la réponse API.

POST /api/admin/users/bulk-import Import en masse depuis Excel / CSV ⬡ Admin ▼

Importe plusieurs utilisateurs depuis un fichier tabulaire. Pour chaque ligne valide : un compte est créé et un email avec les identifiants est envoyé. Les lignes invalides (email/téléphone déjà existant, données manquantes) sont ignorées et listées dans skipped.

Requête (multipart/form-data)

ParamètreTypeContrainte
filerequis file Formats acceptés : xlsx, xls, csv, ods, txt — max 5 MB

Structure du fichier (colonnes dans l'ordre)

Colonne Nom attendu Type Requis Notes
Anametexte✓Nom complet
Bemailemail✓Doit être unique
Cphonetexte✓Doit être unique
Droletexte✓user ou admin
Edomaintexte–Domaine de formation
Fgraduation_yearentier (4 chiffres)–Ex : 2022
Gis_workerbooléen–Valeurs acceptées : 1, true, yes, oui

📎 La première ligne est considérée comme l'en-tête et est ignorée. Les lignes complètement vides sont ignorées silencieusement.

200 Réponse
{
  "type": "success",
  "message": "12 account(s) created, 2 row(s) skipped.",
  "data": {
    "created": 12,
    "skipped": [
      {
        "row": 5,
        "data": { "name": "Alice", "email": "alice@enspm.cm", "phone": "699000000" },
        "reason": ["The email has already been taken."]
      },
      {
        "row": 9,
        "data": { "name": "Bob", "email": "", "phone": "" },
        "reason": ["The email field is required.", "The phone field is required."]
      }
    ]
  }
}
DELETE /api/admin/users/{id} Supprimer un compte utilisateur ⬡ Admin ▼

Supprime définitivement le compte. L'id est passé dans l'URL. Aucun corps de requête requis.

Paramètre d'URL

ParamètreTypeDescription
idrequis integer Identifiant de l'utilisateur à supprimer
200 Réponse
{ "type": "success", "message": "Deleted account", "data": 1 }

🗑️ La suppression est définitive. Il n'existe pas de corbeille ou de désactivation — utilisez cette route avec précaution.

Stages — Vue Admin

Préfixe : /api/admin/internships. Contrairement à la route publique, l'admin voit toutes les offres (actives, suspendues, brouillons) et peut les suspendre/réactiver ou en importer en masse.

GET /api/admin/internships Lister toutes les offres de stage ⬡ Admin ▼

Retourne toutes les offres (tous statuts confondus), paginées par 30. Filtres disponibles en query params.

Query paramTypeDescription
statusoptionnel string Filtrer par statut : active, suspended, draft
titleoptionnel string Recherche par titre (LIKE)
locationoptionnel string Filtrer par lieu (LIKE)
200 Réponse
{
  "data": {
    "data": [
      {
        "id": 1,
        "title": "Stage génie civil",
        "location": "Maroua",
        "organization": "STE SA",
        "status": "active",
        "suspended_by": null,
        "user": { /* auteur de l'offre */ },
        "created_at": "2026-03-10T08:00:00.000000Z"
      }
    ],
    "next_page_url": null
  }
}
GET /api/admin/internships/{id}/suspend Suspendre / Réactiver une offre ⬡ Admin ▼

Route à comportement toggle : si l'offre est active → elle passe à suspended ; si elle est suspended → elle repasse à active. L'ID de l'admin qui suspend est enregistré dans suspended_by (remis à null lors de la réactivation).

Paramètre d'URL

ParamètreTypeDescription
idrequis integer Identifiant de l'offre de stage
200 Suspension (active → suspended)
{ "message": "Offre de stage suspendue avec succès." }
200 Réactivation (suspended → active)
{ "message": "Offre de stage réactivée avec succès." }

💡 Appellez la même route une seconde fois pour inverser l'état. Pas besoin d'une route séparée pour réactiver.

POST /api/admin/internships/import Importer des stages en masse ⬡ Admin ▼

Importe plusieurs offres de stage depuis un fichier Excel ou CSV. Les offres importées sont directement publiées (status = active). L'admin connecté est enregistré comme auteur (user_id) de toutes les offres importées.

Requête (multipart/form-data)

ParamètreTypeContrainte
filerequis file Formats acceptés : xlsx, xls, csv, txt

Structure du fichier — colonnes attendues

La première ligne doit être l'en-tête avec exactement les noms ci-dessous. Les colonnes title, location et organization sont obligatoires.

Nom de colonneTypeRequisContrainte
titletexte✓max 255 caractères
locationtexte✓max 255 caractères
organizationtexte✓max 255 caractères
descriptiontexte–max 5 000 caractères
typetexte–max 100 caractères (ex : présentiel)
linkURL–max 500 caractères, doit être une URL valide
deadlinedate–Format YYYY-MM-DD

⚠️ Si les colonnes obligatoires (title, location, organization) sont absentes de l'en-tête, l'API retourne immédiatement une erreur listant les colonnes manquantes.

200 Réponse
{
  "message": "8 stage(s) importé(s) avec succès.",
  "data": {
    "imported": 8,
    "errors": [
      "Ligne 4 ignorée : The link must be a valid URL.",
      "Ligne 7 ignorée : The title field is required."
    ]
  }
}

Annonces — Vue Admin

Préfixe : /api/admin/announcements. Permet de créer et gérer les annonces officielles de la plateforme (actualités, événements, communiqués). Les annonces peuvent avoir une image, une date de début et une date de fin de visibilité.

GET /api/admin/announcements Lister toutes les annonces ⬡ Admin ▼

Retourne toutes les annonces (tous statuts), paginées par 30. Inclut l'auteur (user). Triées de la plus récente à la plus ancienne.

Query paramTypeDescription
statusoptionnel string Filtrer par statut : active ou suspended
titleoptionnel string Recherche par titre (LIKE)
200 Réponse
{
  "data": {
    "data": [
      {
        "id": 1,
        "title": "Journée portes ouvertes ENSPM 2026",
        "content": "Venez nombreux...",
        "image": "announcements/portes-ouvertes_abc123.jpg",
        "status": "active",
        "start_date": "2026-06-01",
        "end_date": "2026-06-15",
        "suspended_by": null,
        "user": { /* admin auteur */ },
        "created_at": "2026-05-10T09:00:00.000000Z"
      }
    ],
    "next_page_url": null
  }
}

💡 L'URL complète de l'image se construit ainsi : BASE_URL/storage/{image}
Ex : http://your-domain.com/storage/announcements/portes-ouvertes_abc123.jpg

POST /api/admin/announcements Créer une annonce ⬡ Admin ▼

Crée une nouvelle annonce, publiée immédiatement (status = active). Envoyez la requête en multipart/form-data si vous incluez une image.

Corps de la requête (multipart/form-data ou JSON si sans image)

ParamètreTypeContrainteDescription
titlerequis string max:255 Titre de l'annonce
contentrequis string – Corps de l'annonce (texte riche accepté)
imageoptionnel file (image) jpg, jpeg, png, webp Image illustrative de l'annonce
start_dateoptionnel date format YYYY-MM-DD Début de la période de visibilité
end_dateoptionnel date ≥ start_date Fin de la période de visibilité
201 Réponse succès
{
  "type": "success",
  "message": "Annonce créée avec succès.",
  "data": {
    "announcement": {
      "id": 3,
      "title": "Journée portes ouvertes ENSPM 2026",
      "content": "Venez nombreux découvrir...",
      "image": "announcements/portes-ouvertes_6845abc.jpg",
      "status": "active",
      "start_date": "2026-06-01",
      "end_date": "2026-06-15",
      "user": { /* admin auteur */ },
      "created_at": "2026-05-19T10:00:00.000000Z"
    }
  }
}
GET /api/admin/announcements/{id}/suspend Suspendre / Réactiver une annonce ⬡ Admin ▼

Même logique toggle que pour les stages : active → suspended ou suspended → active selon l'état courant.

Paramètre d'URL

ParamètreType
idrequis integer
200 Suspension
{ "message": "Annonce suspendue avec succès." }
200 Réactivation
{ "message": "Annonce réactivée avec succès." }

Annonces — Lecture (Plateforme)

Route accessible à tous les utilisateurs authentifiés (pas besoin d'être admin). Permet au frontend d'afficher les annonces actives de la plateforme.

GET /api/announcements Lister les annonces actives 🔒 Auth ▼

Retourne les annonces dont le statut est active. Inclut l'auteur. Aucun paramètre requis.

200 Réponse
{
  "type": "success",
  "message": "Liste des annonces récupérée avec succès.",
  "data": [
    {
      "id": 1,
      "title": "Journée portes ouvertes ENSPM 2026",
      "content": "Venez nombreux...",
      "image": "announcements/portes-ouvertes_abc123.jpg",
      "status": "active",
      "start_date": "2026-06-01",
      "end_date": "2026-06-15",
      "user": { /* admin auteur */ },
      "created_at": "2026-05-10T09:00:00.000000Z"
    }
  ]
}

💡 Pour afficher l'image : BASE_URL/storage/{image}
Si image est null, aucune image n'a été attachée à l'annonce.

Networking — Membres

Route accessible à tous les utilisateurs authentifiés. Permet de découvrir les autres membres actifs de la plateforme pour faciliter le réseautage entre alumni.

GET /api/networking/members Lister les membres actifs 🔒 Auth ▼

Retourne les utilisateurs actifs (status = true) en excluant les administrateurs (role != admin). Paginé par 20 membres (simple pagination).

200 Réponse
{
  "type": "success",
  "message": "Membres de la plateforme",
  "data": {
    "data": [
      {
        "id": 2,
        "name": "Jean Dupont",
        "email": "jean@enspm.cm",
        "phone": "690000000",
        "role": "user",
        "domain": "Génie Civil",
        "graduation_year": 2022,
        "is_worker": true,
        "profile_picture": "images/profile/jean-dupont.jpg",
        "status": true
      }
    ],
    "next_page_url": "http://your-domain.com/api/networking/members?page=2",
    "prev_page_url": null
  }
}

Règles d'exclusion

ConditionComportement
role = adminExclu — les admins n'apparaissent pas dans la liste networking
status = falseExclu — comptes désactivés/suspendus non visibles
role = user ET status = trueInclus dans les résultats

💡 Pour l'image de profil, construire l'URL complète : BASE_URL/storage/{profile_picture}
Si profile_picture est null, afficher un avatar par défaut côté frontend.
